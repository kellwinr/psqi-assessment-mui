How the Logic Works

This calculator mimics the backend logic of the SPSS/R script provided. Here are the key technical details:

1. The Time Conversion (The "23.5" Rule)

When you select a time in the simulator (e.g., 11:30 PM), the JavaScript converts it to a decimal automatically so the math works.

Input: 11:30 PM
24h Format: 23:30
Decimal Math: 23 + (30/60) = 23.5

This allows the script to calculate Sleep Efficiency (Component 4) by simple subtraction.

2. Sleep Efficiency Formula

The hardest component to calculate manually is Efficiency. The logic follows:

Hrs in Bed = (Wake Decimal - Bed Decimal)
(If result is negative, add 24 hours to handle midnight crossing)

Efficiency % = (Actual Sleep / Hrs in Bed) * 100

3. Scoring Breakdown

  • Global Score Range: 0 to 21
  • Cutoff: A score of 5 or greater indicates "Poor" sleep quality.
